✅Проверить существование ENUM в базе данных
SELECT * FROM pg_type WHERE typname = 'userrole'

✅Если ты хочешь удалить старый ENUM, выполни:
DROP TYPE userrole

✅Чтобы убедиться, что таблицы действительно созданы, зайди в PostgreSQL и выполни:
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public'

✅ Посмотрим, какие типы есть
SELECT typname FROM pg_type

DROP TYPE IF EXISTS roleenum CASCADE;

Как устроены связи в БД
6 основных моделей:

User (Пользователь)

Связан с Goal (один-ко-многим, у пользователя может быть одна цель).
Связан с TestResult (один-ко-многим, у пользователя могут быть несколько тестов).
Связан с Workout (один-ко-многим, пользователь может иметь много тренировок).
Связан с Meal (один-ко-многим, пользователь может иметь много приемов пищи).
Goal (Цель пользователя)

Связана с User (один-ко-многим, одна цель может быть у нескольких пользователей).
TestResult (Результаты теста)

Связаны с User (один-ко-многим, у пользователя может быть несколько тестов).
Workout (Тренировка)

Связана с User (один-ко-многим, у пользователя может быть много тренировок).
FoodItem (Продукты питания)

Связана с Meal (один-ко-многим, один продукт может входить в разные приемы пищи).
Meal (Прием пищи)

Связана с User (один-ко-многим, у пользователя может быть много приемов пищи).
Связана с FoodItem (один-ко-многим, один продукт может быть в нескольких приемах пищи).
